# Skrino

Быстрая скриншотилка для Windows: хоткей → выделение области → редактор (стрелки, текст, фигуры, маркер, размытие, обрезка, счётчик) → «Поделиться» — в локальную папку или на свой сервер по FTP/FTPS/SFTP со ссылкой в буфере обмена.

Нативный Rust + egui: фоновый процесс в трее занимает ~9 МБ памяти, UI запускается только на время работы.

## Сборка

Нужны: [Rust](https://rustup.rs) (stable), MSVC Build Tools, NASM (`winget install NASM.NASM` — для криптографии rustls).

```powershell
cargo build --release
# результат: target\release\skrino.exe (~12 МБ, один файл)
```

## Запуск

Запустите `skrino.exe` — откроется стартовое окно: «Область», «Весь экран», «Открыть файл», внизу «Настройки». При закрытии окна Skrino остаётся в фоне (иконка в трее), и хоткеи продолжают работать.

**Дальше всё через хоткеи и трей:**

| Действие | Как |
|---|---|
| Скриншот области | `Ctrl+Shift+3` (или трей → «Скриншот области») |
| Скриншот всего экрана | `Ctrl+Shift+4` |
| Настройки | трей → «Настройки» |
| Выход из фона | трей → «Выход» |

В оверлее выделения: тянуть мышью — выделить, `Enter`/двойной клик — подтвердить, `Esc` или правый клик — отмена.

В редакторе: `Ctrl+Z`/`Ctrl+Y` — отмена/повтор, `Ctrl+C` — копировать картинку, `Ctrl+S` — сохранить в файл, `Ctrl+Enter` — поделиться, `Ctrl+колесо` — зум.

**Сохранение без вопросов:** первый раз «Сохранить» открывает диалог; после сохранения в тосте есть кнопка «Запомнить папку» — дальше сохранение идёт молча в неё. Изменить — в настройках, раздел «Сохранение».

**Уведомления:** об успешном копировании, сохранении и полученной ссылке приходят системные уведомления Windows (выключаются в настройках). Если закрыть редактор, пока ссылка ещё загружается, — аплоад доработает в фоне и уведомит, когда ссылка окажется в буфере.

## «Поделиться»: локально или на сервер

В настройках, раздел «Поделиться»:

- **В локальную папку** (по умолчанию `Изображения\Skrino`) — картинка сохраняется туда, копируется в буфер, тост показывает путь (клик — открыть в Проводнике). Работает сразу, без настройки.
- **На сервер (FTP/FTPS/SFTP)** — хост, логин, пароль (хранится в системном хранилище Windows, не в конфиге) или SSH-ключ, удалённая папка и шаблон публичной ссылки вида `https://example.com/s/{filename}`. После аплоада готовая ссылка попадает в буфер. Кнопка «Проверить соединение» заливает и удаляет пробный файл.

## Автозапуск

Настройки → «Общие» → «Запускать в фоне при старте системы» (прописывает `skrino.exe --tray` в реестр HKCU\...\Run).

## Флаги командной строки

| Флаг | Что делает |
|---|---|
| *(без флагов)* | стартовое окно |
| `--tray` | фоновый демон: трей + хоткеи, без GPU (~9 МБ) |
| `--capture-region` | сразу выделение области → редактор |
| `--capture-full` | сразу снимок всего экрана → редактор |
| `--open-file` | открыть картинку в редакторе |
| `--settings` | окно настроек |
| `--start` | то же, что без флагов (стартовое окно) |

## Архитектура

Workspace из 4 крейтов:

- `skrino-core` — модель аннотаций, undo/redo, экспорт-рендер (tiny-skia + ab_glyph), без зависимостей от UI
- `skrino-capture` — захват экрана (xcap), мультимонитор, физические пиксели
- `skrino-upload` — FTP/FTPS (suppaftp) и SFTP (russh), генерация неугадываемых имён файлов
- `skrino-app` — egui-приложение: демон трея, оверлей, редактор, настройки

Планы v2: запись видео (ffmpeg-sidecar), кроссплатформенность (capture-слой изолирован под это).
